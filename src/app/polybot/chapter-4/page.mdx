# Chapter 4 — Sizing with the Kelly criterion

The scanner from chapter 3 just found a market where `YES + NO = $1.04`.
You think there&apos;s a real edge. You have $1,000. *How much do you
bet?*

The wrong answers are both the obvious ones:

- **Bet a fixed amount, like $50.** This ignores how big the edge is.
  A 0.5% edge and a 5% edge get the same $50, which is silly — one is
  marginal, the other is enormous.
- **Bet everything.** Even with a 99%-edge sports book you&apos;d go
  broke eventually. Variance makes streaks of losses inevitable; one
  zero ends the game.

The right answer is given by a 1956 paper from John Kelly, an engineer
at Bell Labs who was thinking about how much bandwidth to commit to
noisy signals. The math turns out to be the same.

## The formula

For a binary bet where you buy YES at price `p_market` and you believe
the true probability of YES winning is `p_true`, the optimal fraction
of your bankroll to bet is:

```
b = (1 - p_market) / p_market    # decimal odds: profit per $1 staked
edge = b * p_true - (1 - p_true)
f* = edge / b                    # if edge > 0; else don't bet
```

A worked example. Suppose `YES @ $0.40` and you think the true probability
is 50%:

- `b = 0.60 / 0.40 = 1.5` — you risk $1 to win $1.50
- `edge = 1.5 × 0.5 − 0.5 = 0.25` — positive, so this is a real bet
- `f* = 0.25 / 1.5 = 0.167`

You bet **16.7% of your bankroll**. With $1,000 that&apos;s $167. Not
$50 (too small) and not $1,000 (mathematically suicidal). Kelly says
167 is the number that maximizes the long-run growth rate of your
account.

The interesting comparisons are not the medians — they&apos;re the
**worst-path** and **10th-percentile** columns. Full Kelly has the best
median and the catastrophic tail: in the worst of 1,000 simulated futures,
$1,000 became $27. Half Kelly cuts that tail by 8× ($27 → $212) at the
cost of roughly 30% of the median upside. Quarter Kelly is more
conservative still, and the gap to fixed-dollar narrows.

## The code

```python
import math
import random


def kelly_fraction(p_true: float, price: float) -> float:
    """Optimal fraction of bankroll to bet on YES at given price.

    Args:
        p_true: your estimated true probability of YES winning (0..1)
        price: market price for YES (0..1, exclusive)
    """
    if price <= 0 or price >= 1:
        return 0.0
    b = (1 - price) / price       # profit per dollar staked
    q = 1 - p_true
    edge = b * p_true - q
    if edge <= 0:
        return 0.0
    return edge / b


def simulate_path(
    fraction: float,
    initial: float = 1000.0,
    p_true: float = 0.55,
    price: float = 0.50,
    n_bets: int = 200,
    seed: int = 0,
) -> list[float]:
    """Simulate sequential betting at a constant Kelly multiple."""
    rng = random.Random(seed)
    bankroll = initial
    history = [bankroll]
    b = (1 - price) / price
    for _ in range(n_bets):
        stake = bankroll * fraction
        if rng.random() < p_true:
            bankroll += stake * b   # win: gain b * stake
        else:
            bankroll -= stake       # lose: lose stake
        bankroll = max(bankroll, 0.01)  # avoid log(0) downstream
        history.append(bankroll)
    return history


def max_drawdown(history: list[float]) -> float:
    peak = history[0]
    worst = 0.0
    for value in history:
        peak = max(peak, value)
        if peak > 0:
            drawdown = (peak - value) / peak
            worst = max(worst, drawdown)
    return worst


def fixed_dollar_path(
    stake: float,
    initial: float = 1000.0,
    p_true: float = 0.55,
    price: float = 0.50,
    n_bets: int = 200,
    seed: int = 0,
) -> list[float]:
    rng = random.Random(seed)
    bankroll = initial
    history = [bankroll]
    b = (1 - price) / price
    for _ in range(n_bets):
        actual_stake = min(stake, bankroll)
        if rng.random() < p_true:
            bankroll += actual_stake * b
        else:
            bankroll -= actual_stake
        bankroll = max(bankroll, 0.01)
        history.append(bankroll)
    return history


def run() -> None:
    p_true = 0.55
    price = 0.50
    n_paths = 1000
    n_bets = 200

    f_star = kelly_fraction(p_true, price)
    print(f"Simulating {n_paths} paths × {n_bets} bets each")
    print(f"Edge: true_p={p_true}, price=${price:.2f} → Kelly fraction = {f_star:.3f}")
    print()

    strategies = [
        ("Full Kelly",     lambda s: simulate_path(f_star,        seed=s)),
        ("Half Kelly",     lambda s: simulate_path(f_star * 0.5,  seed=s)),
        ("Quarter Kelly",  lambda s: simulate_path(f_star * 0.25, seed=s)),
        ("Fixed $20",      lambda s: fixed_dollar_path(20.0,      seed=s)),
    ]

    print(f"  {'Strategy':<15} {'Median':>8}  {'10th-pctile':>11}  "
          f"{'Worst path':>13}  {'Max drawdown':>13}")
    print(f"  {'─'*15} {'─'*8}  {'─'*11}  {'─'*13}  {'─'*13}")

    for name, run_fn in strategies:
        histories = [run_fn(seed) for seed in range(n_paths)]
        finals = sorted(h[-1] for h in histories)
        median = finals[n_paths // 2]
        p10 = finals[n_paths // 10]
        worst = finals[0]
        max_dd = max(max_drawdown(h) for h in histories)
        print(f"  {name:<15} ${median:>6,.0f}  ${p10:>9,.0f}  "
              f"${worst:>11,.0f}  {max_dd:>12.0%}")


if __name__ == "__main__":
    run()
```

A few things worth pausing on.

### Kelly maximizes log wealth, not wealth

Full Kelly is provably optimal *if* you&apos;re trying to maximize the
expected logarithm of your final bankroll. That sounds like an arbitrary
choice but isn&apos;t: maximizing expected log wealth is equivalent to
maximizing your *long-run growth rate*. It&apos;s the exact bet size at
which your bankroll grows fastest given infinite repetitions.

It is *not* the size that maximizes your expected final bankroll over
finite horizons, because over finite horizons the variance dominates.
This is the same reason "the lottery has positive EV if the jackpot is
big enough" is technically true and behaviorally irrelevant — your
expected value is great and your median outcome is zero.

### Why half-Kelly

The simulation shows it: at half-Kelly you give up about 30% of your
median winnings (`$3,328 → $2,341`) to cut your worst path by 8×
(`$27 → $212`) and roughly double your 10th-percentile bankroll
(`$447 → $861`). The drawdown numbers are both bad — that&apos;s
inherent to compounding bets at any meaningful Kelly fraction — but
the *terminal* outcomes diverge sharply in the tail.

There&apos;s a deeper reason almost nobody runs full Kelly in
production: **the formula assumes you know `p_true` exactly, and you
don&apos;t**. If you think your edge is 5% but it&apos;s actually 2%,
full-Kelly bets are dramatically oversized — you&apos;re effectively
running 2.5× Kelly on the real edge, which is mathematically guaranteed
to ruin you given enough trades.

Half-Kelly is, conveniently, robust to about a 2× overestimate. You can
think of it as a confidence discount: "I might be wrong about my edge by
a factor of two, so I&apos;ll size as if I am."

### Why Fixed-$20 is closer to Quarter-Kelly than you&apos;d expect

Look at the table: Fixed-$20 has a slightly *lower* median than Quarter
Kelly ($1,440 vs $1,628) but a much *higher* 10th-percentile bankroll
($1,040 vs $988). At $1,000 starting bankroll, $20 is 2% of capital — in
the same ballpark as Quarter Kelly&apos;s 2.5%. The two strategies look
similar precisely because their initial sizes are similar.

The advantage Kelly has over fixed sizing isn&apos;t in any single bet.
It&apos;s in the *next* bet: Kelly automatically reduces stake when
bankroll drops and increases stake when it grows. Over hundreds of
trades, that compounding adjustment dominates. Run the simulation with
`n_bets = 2000` and watch the median spread widen — Quarter Kelly
catches and passes Fixed-$20 because it scales up after winning streaks
while Fixed stays at $20 forever.

## Try this before moving on

Three knobs worth playing with:

1. **Set `p_true = 0.51` and re-run.** A 1% edge halves the Kelly
   fraction roughly, and the median final bankroll collapses. Real
   prediction-market edges are often this small. This is why it&apos;s
   hard to make money even when you&apos;re *right*.
2. **Set `p_true = 0.60` but keep your bets sized at the `0.55` Kelly.**
   You&apos;ve underestimated your edge by 5%. Performance is fine —
   you&apos;re leaving money on the table but not bankrupting. Now
   reverse it: estimate `p_true = 0.60` but the *true* probability is
   `0.55`. Watch full Kelly start losing badly.
3. **Increase `n_bets` to 5000.** Median outcomes diverge dramatically.
   Half-Kelly catches up to and sometimes surpasses full Kelly because
   the drawdown protection prevents any single bad streak from being
   permanent.

## What you just learned

- Kelly tells you *what fraction* of your bankroll to bet given `p_true`
  and `price` — not a fixed dollar amount.
- The formula is three lines of math; the intuition is "bigger edges
  deserve bigger bets, but never bigger than the edge itself."
- Full Kelly maximizes long-run growth rate but has brutal drawdowns and
  is unforgiving of edge-estimation error.
- Half-Kelly is the practical default: ~half the median winnings,
  ~half the drawdown risk, robust to ~2× edge mis-estimation.
- Fixed-dollar sizing looks fine in small samples and falls apart over
  long horizons because it doesn&apos;t compound.
